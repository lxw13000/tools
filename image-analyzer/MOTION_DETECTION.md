# 图片动静态检测 — 算法说明与推荐配置

## 一、三种评分方法

### 1. pHash（感知哈希）

- **原理**：对图片进行 DCT（离散余弦变换）后取低频分量，生成 64 位哈希指纹，比较两张图片的汉明距离
- **计算**：`similarity = 1.0 - (hamming_distance / 64.0)`
- **优点**：抗压缩、抗缩放，计算速度最快
- **缺点**：精度最低，64 位哈希会丢失大量细节信息
- **适用**：快速排除明显不同的图片，作为鲁棒性兜底

### 2. SSIM（结构相似度）

- **原理**：从亮度（luminance）、对比度（contrast）、结构（structure）三个维度衡量两张图片的相似性
- **计算**：图片转灰度 → 统一缩放至 256×256 → `skimage.metrics.structural_similarity(a, b)`
- **优点**：像素级结构对比，对细微画面变化区分度最高
- **缺点**：对压缩噪声和光照变化也敏感，可能产生假阳性
- **适用**：精确衡量画面是否真正发生改变

### 3. Optical Flow（光流法）

- **原理**：使用 Farneback 稠密光流算法计算相邻帧每个像素的位移向量，取位移幅值均值反映运动程度
- **计算**：`cv2.calcOpticalFlowFarneback` → `mean_magnitude = mean(sqrt(fx² + fy²))` → `similarity = 1.0 / (1.0 + mean_magnitude)`
- **优点**：直接测量像素位移，专为运动检测设计，与"动态"定义最相关
- **缺点**：对噪声和光照变化会产生假光流
- **适用**：检测实际物体运动或画面位移

## 二、融合逻辑

三种方法分别计算相邻帧的相似度（0~1），取各自平均值后按权重加权融合：

```
final_score = (w_phash × phash_avg + w_ssim × ssim_avg + w_flow × flow_avg) / (w_phash + w_ssim + w_flow)
```

权重会自动归一化，任一方法失败时自动排除并重新分配剩余权重。

融合评分与双阈值比较，输出三态结果：

```
融合评分 >= 静态阈值  → 静态
融合评分 <  动态阈值  → 动态
介于两者之间          → 人工复核
```

## 三、特性对比

| 方法 | 精度 | 速度 | 抗压缩 | 运动敏感度 | 适用场景 |
|------|------|------|--------|-----------|---------|
| pHash | 低 | 最快 | 强 | 低 | 粗粒度筛选、不同来源图片 |
| SSIM | 高 | 中等 | 中 | 高 | 精确结构对比 |
| Flow | 中 | 较慢 | 弱 | 最高 | 视频截帧、物体运动检测 |

## 四、推荐配置

### 场景 1：通用场景（推荐默认值）

| 参数 | 值 | 理由 |
|------|-----|------|
| SSIM 权重 | **40%** | 结构相似度最精确，对真实画面变化区分度最高 |
| Flow 权重 | **35%** | 直接衡量运动，与"动态"定义最相关 |
| pHash 权重 | **25%** | 作为鲁棒性兜底，防止压缩噪声干扰 SSIM 和 Flow |
| 静态阈值 | **93%** | 95% 偏严格（JPEG 压缩差异可使相同画面降至 94%），93% 更容错 |
| 动态阈值 | **78%** | 低于此值画面已有明显变化，可确信为动态 |
| 复核区间 | 78%~93% | 15% 的缓冲带，覆盖轻微移动/角度微调等模糊地带 |

### 场景 2：视频截帧 / 连续画面检测

| 参数 | 值 | 理由 |
|------|-----|------|
| Flow 权重 | **45%** | 视频帧间运动是核心指标 |
| SSIM 权重 | **35%** | 辅助验证结构变化 |
| pHash 权重 | **20%** | 视频帧质量一致，pHash 参考价值降低 |
| 静态阈值 | **93%** | 同通用场景 |
| 动态阈值 | **78%** | 同通用场景 |

### 场景 3：用户上传图片（来源不一、压缩率不同）

| 参数 | 值 | 理由 |
|------|-----|------|
| pHash 权重 | **40%** | 抗压缩能力最强，不同来源图片需要鲁棒性 |
| SSIM 权重 | **35%** | 仍然是精度最高的方法 |
| Flow 权重 | **25%** | 不同压缩率容易产生假光流 |
| 静态阈值 | **92%** | 来源不同时适当放宽 |
| 动态阈值 | **75%** | 容忍更多压缩差异 |

## 五、阈值设置原则

- **静态阈值不要高于 95%**：JPEG 压缩差异就能导致完全相同的画面 SSIM 降到 0.94~0.96
- **动态阈值不要低于 70%**：低于 70% 只有完全不同的图片才会触发，会漏判大量动态内容
- **复核区间保持 12%~18%**：太窄容易误判，太宽则大量图片需要人工复核，增加运营负担
